### 3. switchTimeRange函数增强

**功能**:
- 支持数字参数（3、7、14、30、60、90）
- 支持字符串参数（'all'）
- 智能匹配按钮状态

**技术实现**:
```javascript
function switchTimeRange(days) {
  currentTimeRange = days;

  // Update button states - 支持字符串和数字
  document.querySelectorAll('.time-range-btn').forEach(btn => {
    const btnDays = btn.dataset.days === 'all' ? 'all' : parseInt(btn.dataset.days);
    if (btnDays === days) {
      btn.classList.add('active');
    } else {
      btn.classList.remove('active');
    }
  });

  // Redraw chart
  updateTrendChart();
}
```

## 💡 设计思考

### 为什么增加这5个时间范围？

**3天**:
- 用户需求：快速查看最新变化
- 场景：日常监控、异常排查
- 频率：高频使用

**14天**:
- 用户需求：2周完整周期（工作日+周末）
- 场景：周期性规律分析
- 频率：中频使用

**60天**:
- 用户需求：2个月中期趋势
- 场景：季度评估、阶段性分析
- 频率：中频使用

**90天**:
- 用户需求：3个月长期趋势
- 场景：战略分析、年度规划
- 频率：低频使用

**全部**:
- 用户需求：完整历史回顾
- 场景：全貌了解、数据审计
- 频率：低频使用

---

### 为什么保持7天为默认？

**原因**:
- 7天是最常用的时间范围
- 既不太短（看不出趋势），也不太长（数据过多）
- 符合用户习惯（一周周期）

**数据支持**: 大多数分析场景都在一周内

---

### 为什么使用flex-wrap而不是下拉菜单？

**原因**:
- 所有选项可见性更好（不需要点击展开）
- 切换更快速（一键点击）
- 视觉扫描更容易

**权衡**: 占用更多空间，但提升了可用性

---

### 为什么按钮间距从12px改为8px？

**原因**:
- 按钮数量增加（2个 → 7个）
- 减少间距更紧凑
- 避免换行过多

**效果**: 更紧凑的布局，减少视觉负担
