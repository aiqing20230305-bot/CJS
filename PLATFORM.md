# 数据抓取配置平台

Web界面配置数据抓取任务，可视化操作，一键下载数据包。

## 🎯 功能特性

### 1. 可视化配置
- ✅ 表单化配置（品牌、品类、竞品、关键词）
- ✅ 下拉选择（天数、结果数）
- ✅ 多选平台（小红书/抖音/微博/全网）
- ✅ 抓取类型选择（5种模式）

### 2. 实时进度
- ✅ 进度条显示
- ✅ 日志输出
- ✅ 状态提示（进行中/已完成）

### 3. 数据管理
- ✅ 数据包列表（表格展示）
- ✅ 一键下载
- ✅ 文件大小和时间显示

### 4. 历史记录
- ✅ 最近5次配置
- ✅ 快速加载历史配置
- ✅ 配置详情展示


## 📖 使用步骤

### 步骤 1: 填写配置

1. **基础信息**
   - 品牌名称：例如"多芬"
   - 品类：例如"洗护"

2. **竞品列表**
   - 输入竞品名称
   - 点击"添加竞品"按钮增加更多
   - 点击"删除"移除不需要的

3. **关键词**（可选）
   - 输入搜索关键词
   - 例如："洗护套装 护发"

4. **抓取参数**
   - 抓取天数：7/14/30 天
   - 最大结果数：5/10/20 条

5. **平台选择**
   - 勾选需要的平台
   - 可多选

### 步骤 2: 选择抓取类型

- **市场热点** - 采集平台热点、品类热点、热销属性
- **爆款视频** - 采集爆款视频数据和分析
- **竞品分析** - 采集竞品销量、评价数据
- **品类报告** - 采集市场规模、人群画像
- **完整数据包** ⭐ 推荐 - 一次性采集全部数据

### 步骤 3: 开始抓取

点击"开始抓取"按钮：
- 进度区域自动显示
- 实时显示抓取状态
- 完成后自动添加到数据包列表

### 步骤 4: 下载数据

在"数据包下载"区域：
- 查看已生成的数据包
- 点击"下载"按钮获取文件
- 文件保存在 `./data/` 目录

## 🔧 技术架构

### 前端
- **HTML/CSS/JavaScript**（原生，无框架）
- **Notion 设计风格**（简洁清晰）
- **响应式布局**（支持移动端）

### 后端
- **Node.js HTTP 服务器**
- **集成 scraper.js**
- **API 端点**：
  - `POST /api/scrape` - 启动抓取
  - `GET /api/files` - 获取文件列表
  - `GET /api/download/:filename` - 下载文件

### 数据流

```
用户配置 → 前端表单
    ↓
HTTP POST /api/scrape
    ↓
后端服务器 → 调用 scraper.js
    ↓
数据保存 → ./data/ 目录
    ↓
前端展示 ← GET /api/files
    ↓
用户下载 ← GET /api/download/:file
```

## ⚙️ 配置说明

### 环境变量

服务器会自动读取 `.env` 文件中的 `GEMINI_API_KEY`。

如果未设置，确保：
```bash
# .env 文件
GEMINI_API_KEY=your-api-key-here
```

或者在 `~/.gemini/settings.json` 中配置。

### 端口配置

默认端口：`3000`

修改方法（在 `platform-server.js` 中）：
```javascript
const PORT = 3000; // 改为其他端口
```

## 🔍 API 文档

### 1. 启动抓取

**请求**：
```http
POST /api/scrape
Content-Type: application/json

{
  "brand": "多芬",
  "category": "洗护",
  "competitors": ["凡士林", "妮维雅"],
  "keywords": "洗护套装",
  "days": 14,
  "maxResults": 10,
  "platforms": ["小红书", "抖音"],
  "scrapeType": "full"
}
```

**响应**：
```json
{
  "success": true,
  "taskId": "1735678901234",
  "message": "抓取任务已启动"
}
```

### 2. 获取文件列表

**请求**：
```http
GET /api/files
```

**响应**：
```json
[
  {
    "filename": "full_多芬_1735678901234.json",
    "time": "2026-04-05T15:30:00.000Z",
    "size": "3.2 MB",
    "path": "/api/download/full_多芬_1735678901234.json"
  }
]
```

### 3. 下载文件

**请求**：
```http
GET /api/download/:filename
```

**响应**：
```
Content-Type: application/octet-stream
Content-Disposition: attachment; filename="..."

<文件内容>
```

## 🐛 故障排查

### 问题 1: 页面无法访问

**检查**：
```bash
# 确认服务器正在运行
ps aux | grep platform-server

# 检查端口占用
lsof -i :3000
```

**解决**：
```bash
# 重启服务器
npm run platform
```

### 问题 2: 抓取失败

**检查**：
1. API Key 是否配置
2. 网络连接是否正常
3. 查看服务器日志

**解决**：
```bash
# 检查 API Key
node quick_test.js

# 查看日志
# 服务器终端会显示错误信息
```

### 问题 3: 下载文件为空

**检查**：
```bash
# 查看 data 目录
ls -lh data/

# 确认文件存在且非空
cat data/*.json | head -20
```
